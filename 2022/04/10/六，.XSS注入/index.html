xss简介

xss是跨站脚本攻击(Cross Site Scripting)的简写。它的形成与sql注入类似，是由于开发过程中未对用户提交的恶意输入过滤直接输出到界面。通常攻击者可以利用其执行JavaScript，HTML等代码。xss与不同于sql，它不一定需要与数据库交互。

xss通常可以分为反射型xss（非持久化），存储型xss（持久化），DOM型xss

漏洞发现

如果用户的输入会添加在前端代码，就有可能存在xss注入

如url的关键词，用户名密码输入等等地方可能存在反射型xss漏洞。这种漏洞的特点是即时，但是主动出击（诱导目标点击具有反射型xss漏洞的网址）

在像留言板评论区这些地方可能存在存储型xss漏洞。这种漏洞的特点是可以长久危害，所有人访问网站都能看到留下的网址，可以说是被动等待上钩

基于 DOM 的 XSS 漏洞通常出现在 JavaScript 从攻击者可控制的来源（例如 URL）获取数据并将其传递到支持动态代码执行的接收器（例如eval()或innerHTML. 这使攻击者能够执行恶意 JavaScript，这通常允许他们劫持其他用户的帐户。

​ ps：文档对象模型 (DOM) 是网页浏览器对页面元素的分层表示。网站可以使用 JavaScript 来操作 DOM 的节点和对象，以及它们的属性。DOM 操作本身不是问题。事实上，它是现代网站运作方式不可或缺的一部分。但是，不安全地处理数据的 JavaScript 可能会引发各种攻击。当网站包含的 JavaScript 获取攻击者可控制的值（称为源）并将其传递给危险函数（称为接收器）时，就会出现基于 DOM 的漏洞

一些浏览器查看图片 Exif 信息的插件，如果图片里面含有 XSS 代码，这些插件在展现信息渲染的时候就会被执行，导致在本不存在漏洞的网页中“制造”出 XSS 漏洞。

漏洞利用

利用跨站点脚本漏洞的攻击者通常能够：

  • 冒充或伪装成受害者用户。
  • 执行用户能够执行的任何操作。
  • 读取用户能够访问的任何数据。
  • 捕获用户的登录凭据。
  • 对网站进行虚拟污损。
  • 将木马功能注入网站
